# HA Automation Maker

Home Assistant용 **한국어 우선 자동화 애드온**입니다. 두 가지 방식으로 자동화를
만들 수 있습니다.

## 루틴 — 자연어로 규칙 만들기 (기본 화면)

"화장실은 5분 동안 움직임이 없으면 환풍기와 조명을 꺼줘"처럼 평소 말투로 한 문장을
쓰면, 애드온이 스스로 뜻을 해석해 트리거·조건·액션으로 바꿉니다. 해석이 애매한
부분은 확인 카드에서 사람이 후보 중 하나를 짚어 확정하면 됩니다.

이렇게 만든 규칙은 Home Assistant의 `automations.yaml`을 전혀 건드리지 않고,
애드온이 직접 상태를 구독하고 서비스 호출로 실행하는 **자체 규칙 엔진**으로
동작합니다. 규칙·설정·실행 기록은 애드온 데이터 폴더(`/data`)에 저장되어 Home
Assistant 백업에 함께 포함됩니다.

## HA 자동화 — 기존 카드 편집기 (별도 탭)

방(거실·안방 등)과 센서 카테고리(조명/감지기/환경 센서/난방 등) 단위로 엔티티를
탐색하며 트리거·조건·액션을 카드로 조합해, Home Assistant 표준 자동화
(`automations.yaml` 기반, `automation.*` 엔티티)를 만드는 화면입니다. 한글 초성
검색, 자연어 요약, YAML 미리보기를 그대로 지원합니다.

## 주요 기능

- 자연어 문장으로 루틴 등록 + 해석 확인 카드(칩 하이라이트, 불확실한 부분만 확정)
- 전역 시간 변수(새벽/아침/낮/저녁/밤 시간대, 계절, 평일/주말/공휴일) 기반 조건 해석,
  시간대 경계는 설정 화면에서 조정 가능
- 자체 규칙 엔진 — HA `automations.yaml` 미사용, `/data`에 저장(HA 백업에 자동 포함)
- 선택적 AI 해석 보조(애드온 옵션에 Anthropic API 키를 넣은 경우에만 동작)
- 설정 화면: 시간대 경계·사람 이름 매핑·모드(씬/스크립트) 매핑·엔티티 별칭 관리
- 기존 트리거·조건·액션 카드 편집기(HA 자동화 탭): 방 → 카테고리 → 엔티티 3단 피커,
  초성 검색, 자연어 요약, YAML 미리보기, 안전 관련 액션 확인 경고

자세한 사용법은 **문서(Documentation)** 탭을 참고하세요.

이 애드온은 Home Assistant Core API(`homeassistant_api`)만 사용합니다. AI 해석
보조를 켜지 않으면 외부로 나가는 요청이 없습니다. Ingress를 통해 사이드바
패널로 노출됩니다.
