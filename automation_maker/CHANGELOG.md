# 변경 이력

## 3.2.0

**자연어 자동화 정확도 대폭 향상 — 훨씬 다양한 표현을 이제 로컬에서(무료·오프라인)
이해합니다.** SmartThings·Fibaro·Homey·HomeKit 등 주요 스마트홈 플랫폼의 자동화 문법을
리서치해, 그동안 못 읽던 표현을 규칙 파서에 대거 추가하고 실제로 실행되는 엔진 노드를
새로 만들었습니다. 정직 측정(라이브러리에 쓰지 않은 별도 평가셋 520문장) 기준 로컬 파서
정확도가 21.7% → **66.0%**로 올랐습니다.

새로 이해하는 표현(모두 확인 카드에 한국어로 풀어서 보여줍니다):

- **태양 이벤트**: "해 지면", "해 지기 30분 전에", "해 뜰 때", "어두워지면", "밤사이엔"
  → 일출·일몰 기준으로 실제 스케줄되어 발화합니다. *일출/일몰 시각은 위치 설정을 씁니다
  (기본값 서울; 다른 지역은 애드온 설정에서 위도·경도를 지정).*
- **요일·달력**: "월수금 아침 7시에", "주말 빼고", "매달 1일에", "말일에", "격주 금요일에".
- **주기·반복**: "30분마다", "두 시간에 한 번씩", "3번 깜빡여줘", "…할 때까지 반복".
- **재실 인원**: "집에 아무도 없으면", "누구든 오면", "모두 귀가하면", "마지막 사람이 나가면"
  (특정인 지정·"N분 동안 없으면"도 지원).
- **상태 구분**: "켜지면"(변화=트리거)과 "켜져 있으면"(상태=조건)을 구분합니다.
- **수치 방향**: "25도 넘어가면", "10% 밑으로 떨어지면", "A와 B 사이면".
- **조명 세부**: "빨간색으로", "전구색/주광색으로", "좀 더 밝게", "서서히", "반대로(토글)".
- **조건 분기**: "…면 X, 아니면 Y", "평일엔 켜고 주말엔 꺼".
- **한정 지속·복원**: "3분만 켰다 꺼".
- **알림 메시지**: "'문 열렸어요'라고 알려줘"(폰/스피커 지정).
- **자연스러운 말투**: 한글 수사("일곱 시 반"), 다양한 어미(-거든/-자마자/-는 순간),
  존댓말(주무시면), 완곡표현(좀 ~해줘), 후치 어순("불 꺼줘, 문 열리면")도 정규화해 이해합니다.

- **AI(구독 CLI) 학습은 롱테일 전용으로 최소화**: 파서와 패턴 매칭이 **모두** 실패한
  문장에서만, 사용자가 명시적으로 선택할 때 AI가 1회 표준화하고, 그 결과를 로컬 라이브러리에
  증류(학습)해 **이후 같은/비슷한 문장은 AI 없이** 처리합니다. 대부분의 자동화는 AI 호출
  없이 로컬에서 즉시 만들어집니다.
- **안전장치**: 새 표현이 늘었어도 금지문("가스밸브 열지 마")은 정반대 동작을 절대 만들지
  않고, `homeassistant.stop`/`restart` 같은 관리자 서비스는 저장·실행을 차단합니다. 재실
  기반 규칙은 상태가 유지되는 동안 중복 발화하지 않습니다.
- **HA 자동화 탭 연동**: 위 신규 규칙도 Home Assistant 네이티브 자동화로 내보낼 수 있습니다
  (일출·일몰, 요일, 반복 등).

## 3.1.0

**하이브리드 패턴 매칭 + AI 학습(선택 기능) 추가.** 로컬 규칙 파서가 해석하지
못한 문장을, 내장 패턴 라이브러리로 흡수하거나(무료·오프라인) AI로 표준 문형
으로 바꿔 애드온이 배우게 할 수 있습니다.

- **하이브리드 패턴 매칭(자동, 무료)**: 애드온에 자주 쓰이는 표준 문형
  라이브러리를 내장했습니다. 로컬 파서가 실패한 문장에 한해, AI를 부르기
  전에 이 라이브러리와 문형 골격을 대조해(어휘를 태그열로 바꿔 정렬) 맞으면
  정답 구조에 값을 끼워 규칙을 만듭니다. AI 호출·인터넷 연결이 필요 없고,
  파서가 이미 해석한 문장은 절대 덮어쓰지 않으며(회귀 없음), 액션 극성(켜/끄기)
  까지 일치할 때만 채택합니다. 확인 카드에 출처가 "패턴 매칭"으로 표시됩니다.
- **AI로 분석해서 배우기(학습, 기본 꺼짐)**: 파서·패턴 매칭이 모두 실패한
  문장의 확인 카드에 **[AI로 분석해서 배우기]** 버튼을 추가했습니다. AI가
  문장을 표준 문형으로 다시 써서 규칙 미리보기를 만들고, 사용자가 승인하면
  (원문 → 규칙) 매핑을 학습합니다. 이후 같은/비슷한 문장은 AI 없이 로컬에서
  즉시 처리됩니다(출처 "학습됨"). 표준 문형 변환에는 기존 LLM 해석 백엔드
  (API 키 / 구독 CLI)가 준비돼 있어야 합니다.
- **안전장치**: AI에게는 인벤토리에 있는 방·기기·모드만 쓰도록 지시하고,
  변환된 문장이 원문에 없던 대상을 새로 지어냈는지 검사해 저장을 막습니다
  (환각 방어). 저장 전 엔티티 실존을 재검증하고, **동작과 대상 기기 종류가
  맞지 않거나(예: 조명에 환풍기 켜기) 대상이 비어 있으면 저장을 막으며**,
  최종 저장은 사용자 승인이 있어야 이뤄집니다. 학습 항목이 참조하는 기기가
  사라지면 재기동 시 자동 정리하되, 인벤토리 로드 자체가 실패한 경우(HA 부팅
  중)에는 학습 데이터를 건드리지 않습니다.
- **설정 화면**에 "AI 학습" on/off 스위치와, 배운 항목(원문·표준 문형·사용
  횟수)을 보고 삭제하는 **학습된 패턴** 목록을 추가했습니다.
- 학습 항목은 `/data/learned_patterns.yaml`에 저장되어 Home Assistant 백업에
  포함됩니다.
- API 확장: `POST api/v2/learn`(문장 분석·미리보기), `POST api/v2/learn/confirm`
  (학습 저장), `GET api/v2/learned`·`DELETE api/v2/learned/{id}`(학습 항목
  목록·삭제). 설정에 `learn.enabled` 항목 추가. `POST api/v2/parse` 응답에
  해석 출처를 나타내는 `source`(`rule`/`pattern`/`learned`) 필드 추가.

## 3.0.0

**상태 모드 · 다중 규칙 문장 · 수동 단어 매핑 · Claude CLI 연동 추가.** 루틴
(자연어 규칙) 엔진이 상태를 가진 변수, 한 문장 다중 규칙, 수동 매핑 편집기,
그리고 API 키 없이도 쓸 수 있는 Claude 구독 CLI 연동을 지원합니다.

- **상태 모드**: "슬립 모드"처럼 켜짐/꺼짐 상태를 계속 기억하는 모드 변수를
  트리거("슬립모드가 켜지면")·조건("…슬립모드이고…")으로 쓸 수 있습니다.
  **설정 → 모드 관리**에서 모드 추가·삭제, 초기값, 켜질 때 실행할 씬 지정,
  저장된 모드의 현재 상태 수동 켜기/끄기를 지원합니다. 문장이 참조하는 새
  모드는 저장 시 초기값 꺼짐으로 자동 등록됩니다.
- **한 문장에 여러 규칙 담기**: "~해주고 …하면 …해줘"처럼 (조건 → 액션) 쌍이
  여러 개인 문장을 하나의 카드로 저장하되, 내부적으로는 규칙 여러 개(서브룰)로
  나눠 각각 독립적으로 발화시킵니다. 뒤쪽 규칙에서 대상을 생략하면 앞
  규칙에서 마지막으로 언급한 엔티티를 이어받습니다(컨텍스트 상속). 루틴
  목록에는 "규칙 N개"로 표시됩니다.
- **수동 단어 매핑("직접 지정")**: 파서가 이해하지 못한 문장을 단어(토큰)
  단위로 쪼개, 각 단어에 역할(트리거/조건/액션 대상, 상태값, 수치 조건,
  지속시간, 시간대, 모드, 요일, 계절, 값, 동작, 규칙 나눔)을 직접 지정해
  규칙을 만드는 편집기를 해석 확인 카드에 추가했습니다. "초안 채우기"로
  파서 힌트를 자동으로 채운 뒤 수정할 수 있습니다.
- **LLM 해석 백엔드 확장**: 기존 "선택적 AI 해석 보조"가 **끄기 / API 키 /
  Claude 구독 CLI** 3가지 선택지로 확장됐습니다. 구독 CLI는 애드온 이미지에
  Claude Code CLI를 내장해(이미지 용량 약 250MB 증가) `claude setup-token`으로
  발급한 구독 토큰으로 호출합니다. API 키가 설정되어 있으면 CLI 인증 규칙상
  구독 토큰보다 우선하므로, 구독 CLI를 쓰려면 API 키 칸을 비워 두는 것을
  권장합니다.
- 골든 문장 예시 추가:
  - "슬립모드가 켜지면 집의 모든 조명을 꺼줘"
  - "새벽에 슬립모드이고 거실 모션이 작동하면 거실조명을 10% 켜주고 3분 동안
    모션이 없으면 꺼줘" → 규칙 2개로 자동 분해
- API 확장: `GET/POST api/v2/modes`(모드 목록·수동 토글), `POST
  api/v2/tokenize`·`POST api/v2/build`(수동 단어 매핑).
- 애드온 옵션에 `llm_backend`(끄기/API 키/구독 CLI), `claude_code_oauth_token`
  (구독 토큰) 항목 추가.

## 2.0.0

**자연어 규칙 엔진 추가.** Home Assistant `automations.yaml`을 쓰지 않고 애드온이
직접 규칙을 해석하고 실행하는 자체 엔진이 새로 생겼습니다. 기존 트리거·조건·액션
카드 편집기는 사라지지 않고 **"HA 자동화"** 탭으로 그대로 남아 있습니다.

- 한국어 문장으로 규칙을 등록하는 **루틴** 화면 추가(기본 탭). 예:
  - "화장실은 5분 동안 움직임이 없으면 환풍기와 조명을 꺼줘"
  - "나와 와이프가 집에 도착하면 현관 조명을 켜줘"
  - "여름에 거실 온도가 28도 이상이면 에어컨을 켜줘"
  - "안방은 30분 동안 움직임이 없으면 슬립 모드로 바꿔"
  - "거실 조명은 새벽시간에 거실에 움직임이 있으면 10%로 켜줘"
- 문장을 트리거·조건·액션으로 해석한 결과를 사람이 눈으로 확인하고 애매한 부분만
  짚어 확정하는 **해석 확인 카드** 추가(칩 하이라이트 + 후보 선택 + 엔티티 피커).
- 자체 규칙 엔진: Home Assistant는 WebSocket으로 상태를 구독하고 서비스 호출로
  기기를 제어하는 용도로만 쓰고, `automations.yaml`은 전혀 건드리지 않습니다.
- 규칙·설정·실행 기록을 애드온 영속 데이터 폴더(`/data`)에 저장 → Home Assistant
  백업에 자동으로 포함됩니다.
- 전역 시간 변수 도입: 새벽/아침/낮/저녁/밤 시간대, 계절(봄/여름/가을/겨울),
  요일 구분(평일/주말/공휴일). "새벽시간에", "여름에" 같은 표현을 이 변수로
  해석합니다. 시간대 경계는 설정 화면에서 조정할 수 있습니다.
- **설정** 화면 추가: 시간대 경계, 사람 이름 매핑(나/와이프 + 추가), 모드(씬·
  스크립트) 매핑, 엔티티 별칭 관리.
- 선택적 **AI 해석 보조**: 애드온 옵션에 Anthropic API 키를 입력한 경우에만,
  로컬 파서가 확신하지 못하는 문장에 한해 보조로 다시 해석합니다. 키를 넣지
  않으면 완전히 로컬 파서만으로 동작하며 외부로 나가는 요청이 없습니다.
- 헤더에 엔진 연결 상태 표시(연결됨/끊김)와 현재 시간대 라벨 추가.
- 오류가 3회 연속 발생한 규칙은 자동으로 꺼지고("오류로 꺼짐" 배지), 토글을
  껐다 켜면 오류 상태가 초기화되고 다시 활성화됩니다.
- 애드온 옵션에 로그 레벨, Anthropic API 키(선택) 항목 추가.

## 1.0.0

첫 릴리스.

- 방 → 카테고리 → 엔티티 3단 피커, 한글 초성 검색 지원
- 트리거 카테고리 위저드(시간/사람 이동/센서 감지/수치 변화)로 새 자동화 시작
- 트리거·조건·액션 카드 스택 편집기 (`~할 때` / `~인 동안` / `실행`)
- 트리거 8종(state, numeric_state, time, time_pattern, sun, zone, template,
  homeassistant), 조건(state, numeric_state, time, sun, zone, template,
  trigger, and/or/not 조합), 액션 10종(service, delay, wait_template,
  wait_for_trigger, condition, choose, if, repeat, parallel, stop) 지원
- 저장 전 한국어 자연어 요약 및 YAML 미리보기
- 자동화 목록 조회, 생성/수정/삭제, 켜기·끄기 토글, 수동 실행
- 가스밸브·잠금장치 등 안전 관련 액션 포함 시 저장 확인 경고 표시
- Home Assistant Ingress 전용 접근 제어(게이트웨이 IP만 허용)
- `DEV_MODE=1`로 한국 아파트 가정 목데이터를 이용한 로컬 개발 모드 지원
